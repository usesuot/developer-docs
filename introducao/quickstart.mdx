Este quickstart mostra o **modelo mental** do Fiscal SDK com CT-e. Você não precisa de token Composer para acompanhar. Quando for instalar no seu projeto, vá para [Instalação](/fiscal/instalacao).

<Info>
  **Pré-requisitos para só ler e entender:** nenhum.  
  **Para integrar no seu app:** PHP 8.4+, Composer 2, licença e token — [Instalação](/fiscal/instalacao).
</Info>

## O que você vai aprender

1. API padrão: `$cte->…->handle()`
2. Rejeição fiscal é `Result`, não exception
3. O host persiste chave/protocolo/XML e decide retry

## Emitir com `$cte->issue()`

```php
use Suot\Fiscal\Cte\Application\Cte;
use Suot\Fiscal\Core\Exception\InfrastructureException;
use Suot\Fiscal\Core\Result\RetryMode;
use Suot\Fiscal\Core\ValueObject\Uf;
// … Data objects (CteEmitter, CteMunicipality, …)

/** @var Cte $cte */
try {
    $result = $cte->issue()
        ->forOperation('ot-000123')
        ->inHomologation()
        ->numbered('1', '12345')
        ->issuedIn(new Uf('RS'))
        ->withOperation('5353', 'PRESTACAO DE SERVICO DE TRANSPORTE')
        ->dispatchedFrom($dispatchMunicipality)
        ->issuedBy($emitter)
        ->from($origin)
        ->to($destination)
        ->forShipper($shipper)
        ->deliverTo($receiver)
        ->carrying($cargo)
        ->charging($freight)
        ->taxedBy($tax)
        ->linkedToDocument($linkedNfe)
        ->underRntrc('12345678')
        ->handle();
} catch (InfrastructureException $e) {
    $failure = $e->toFailure('issue');
    // Host: alertar; filar retry conforme $failure->retryPolicy (timeout ambíguo → QueryBeforeRetry)
    throw $e;
}

if ($result->isAuthorized()) {
    // Host: persistir accessKey, protocolNumber, submittedDocumentXml, responseXml, receivedAt, operationId
}

if ($result->isRejected()) {
    $failure = $result->failure();
    $policy = $result->retryPolicy();
    match ($policy?->mode) {
        RetryMode::AfterCorrection => /* remontar issue() com dados corrigidos + novo forOperation */,
        RetryMode::QueryBeforeRetry => /* byAccessKey() antes de reemitir */,
        RetryMode::AfterDelay, RetryMode::Immediate => /* filar conforme política */,
        RetryMode::Never, null => /* definitivo; não reenviar */,
    };
}

if ($result->isProcessing()) {
    // Host: consultar por chave/protocolo; não emitir de novo às cegas
}
```

Passo a passo com cadeia tipada completa: [Emissão de CT-e](/fiscal/cte/emissao).

## Pré-voo antes de emitir

```php
use Suot\Fiscal\Core\Exception\InfrastructureException;
use Suot\Fiscal\Core\ValueObject\Uf;

try {
    $validation = $cte->validate()
        ->issuerSetup()
        // … parâmetros do emitente …
        ->handle();
} catch (InfrastructureException $e) {
    // Host: alertar falha de certificado/local; não emitir
    throw $e;
}

if (! $validation->isValid()) {
    $failure = $validation->failure();
    // Host: corrigir dados (AfterCorrection típico); não transmitir
}

try {
    $status = $cte->serviceStatus()
        ->forOperation('preflight-1')
        ->inHomologation()
        ->issuerUf(new Uf('RS'))
        ->handle();
} catch (InfrastructureException $e) {
    $failure = $e->toFailure('service_status');
    // Host: alertar / filar retry; não emitir às cegas
    throw $e;
}

if (! $status->isOperational()) {
    $failure = $status->failure();
    // Host: aguardar ou filar conforme retryPolicy(); não emitir
}
```
