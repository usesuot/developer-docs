Toda operação pública de CT-e devolve um `*Result` tipado. Rejeição fiscal **não** é exception.

## Estados

| Situação | Como tratar |
|---|---|
| Autorizado / registrado | `isAuthorized()` (emissão) ou `!isRejected()` / `isFound()` / `isValid()` conforme o Result |
| Rejeição SEFAZ / fiscal | `isRejected()` → `failure()` → `retryPolicy()` |
| Processando / ambíguo | `isProcessing()` ou consultar (`QueryBeforeRetry`) |
| Infra (timeout, rede, cert) | `InfrastructureException` — `try/catch` + `toFailure()` |

## Exemplo (emissão)

```php
use Suot\Fiscal\Core\Exception\InfrastructureException;
use Suot\Fiscal\Core\Result\RetryMode;

try {
    $result = $cte->issue()
        ->forOperation('ot-123')
        ->inHomologation()
        // … cadeia …
        ->handle();
} catch (InfrastructureException $e) {
    $failure = $e->toFailure('issue');
    // Host: alertar; filar retry conforme $failure->retryPolicy (timeout ambíguo → QueryBeforeRetry)
    throw $e;
}

if ($result->isAuthorized()) {
    // Host: persistir accessKey, protocolNumber, submittedDocumentXml, responseXml, receivedAt, operationId
}

if ($result->isRejected()) {
    $failure = $result->failure();
    $policy = $result->retryPolicy();
    match ($policy?->mode) {
        RetryMode::AfterCorrection => /* remontar issue() com dados corrigidos + novo forOperation */,
        RetryMode::QueryBeforeRetry => /* byAccessKey() antes de reemitir */,
        RetryMode::AfterDelay, RetryMode::Immediate => /* filar conforme política */,
        RetryMode::Never, null => /* definitivo; não reenviar */,
    };
}

if ($result->isProcessing()) {
    // Host: consultar por chave/protocolo; não emitir de novo às cegas
}
```

O mesmo envelope (`try/catch`, `failure()`, `retryPolicy()`, ações do host) aparece em cancelamento, eventos, EPEC, contingência, consultas e `downloadProc`.

## O que o host faz

- Persistir artefatos do Result ([CT-e → Resultados](/fiscal/cte/resultados))
- Executar retry conforme [Retry](/fiscal/retry) — o SDK **não** reenvia sozinho
- Corrigir dados remontando `$cte->issue()…` com novo `forOperation` quando `AfterCorrection`
