O **BP-e** (modelo 63) cobre bilhete de passagem rodoviária. O client público é `Suot\Fiscal\Bpe\Application\Bpe` via `$bpe->…->handle()`.

<Warning>
Capacidades fiscais só são production-ready após schemas importados, fixtures, ciclo de vida completo e evidência de homologação. O ciclo mínimo (status, emissão, consulta, cancelamento) está implementado, mas a homologação de emissão ainda **não** atingiu `cStat 100` — bloqueio atual `203` (emitente não habilitado para BP-e). Status do serviço homologado com `cStat 107` (2026-07-16).
</Warning>

## Pacote e instalação

```bash
composer require suot/fiscal-core suot/fiscal-sefaz suot/fiscal-bpe
```

Vendas, reservas, assentos e embarque ficam no host — o SDK trata apenas o documento fiscal.
