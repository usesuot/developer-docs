O pacote `usesuot/fiscal-cte` é framework-agnostic. Não há um “FiscalConfig” único obrigatório na API pública: a aplicação **liga** adapters de certificado e comunicação SEFAZ aos Handlers e passa-os ao client `Cte`.

## O que o host precisa

1. Certificado A1 (PKCS#12) acessível em runtime
2. Ambiente `homologation` / `production` (flag explícita para produção)
3. Handlers CQRS construídos com o pipeline interno (`Operation/` é `@internal`)
4. Instância de `Suot\Fiscal\Cte\Application\Cte` com esses Handlers

```php
use Suot\Fiscal\Cte\Application\Cte;

// Pseudocódigo: sua fábrica / container monta os Handlers
$cte = new Cte(
    $issueHandler,
    $cancelHandler,
    // … demais Handlers + CteValidate
);

$result = $cte->serviceStatus()/* … */->handle();
```

## Laravel

Prefira [Laravel Bridge](/fiscal/laravel-bridge): Service Provider registra certificado, gateway, `Cte` e aliases. Publique `config/fiscal.php` e use variáveis `FISCAL_*`.

## Produção

- Não chame `inProduction()` sem `FISCAL_ALLOW_PRODUCTION` (ou equivalente no host) e homologação prévia
- Schemas e endpoints vêm dos pacotes — não baixe XSD em tempo de emissão
