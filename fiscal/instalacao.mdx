## Requisitos

1. **PHP 8.4+** e **Composer 2**
2. Extensões: `openssl`, `soap`, `curl`, `mbstring`, `dom`
3. **Só na hora de baixar pacotes:** organização + licença + [token Composer](/platform/tokens-composer)

Ler a documentação **não** exige conta. O registry privado exige autenticação.

## Instalar

<Steps>
  <Step title="Registry">
    ```bash
    composer config repositories.suot composer https://packages.suot.dev
    ```
  </Step>

  <Step title="Autenticação">
    ```bash
    export COMPOSER_AUTH='{"http-basic":{"packages.suot.dev":{"username":"token","password":"SEU_TOKEN"}}}'
    ```

    <Warning>
    Não versione tokens. Em CI, use secret `COMPOSER_AUTH`.
    </Warning>
  </Step>

  <Step title="Require">
    CT-e:

    ```bash
    composer require suot/fiscal-core suot/fiscal-sefaz suot/fiscal-cte
    ```

    CT-e OS (fretamento / modelo 67):

    ```bash
    composer require suot/fiscal-core suot/fiscal-sefaz suot/fiscal-cte-os
    ```

    Laravel (opcional):

    ```bash
    composer require suot/fiscal-laravel
    ```
  </Step>

  <Step title="Verificar">
    ```php
    <?php
    require __DIR__ . '/vendor/autoload.php';

    use Suot\Fiscal\Core\ValueObject\Cnpj;

    echo Cnpj::class, PHP_EOL;
    ```
  </Step>
</Steps>

## CI

Injete `COMPOSER_AUTH` como secret. Token dedicado de CI — não compartilhe token pessoal.

## Produção vs avaliação

| Avaliação | Produção |
|---|---|
| Docs públicas + snippets inline (sem SEFAZ) | Pacotes versionados + certificado + autorizador real |
| Estudo da API sem instalar | Homologação SEFAZ antes de `inProduction()` |

<Note>
Vencimento de atualização **não** é kill switch no runtime já instalado. Sem renovação você deixa de receber updates e suporte — o software instalado continua.
</Note>
