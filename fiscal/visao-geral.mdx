Biblioteca PHP **framework-agnostic** para documentos fiscais eletrônicos de transporte. Comece pelo CT-e; MDF-e e demais famílias seguem o mesmo modelo de Result.

<Note>
Ler a documentação não exige licença. Instalar via Composer a partir de `packages.suot.dev` exige organização, licença e token.
</Note>

## Pacotes

| Pacote | Função |
|---|---|
| `usesuot/fiscal-core` | Tipos, VOs, Result, retry |
| `usesuot/fiscal-sefaz` | Autorizadores e comunicação SEFAZ |
| `usesuot/fiscal-cte` | CT-e (API pública `Application\Cte`) |
| `usesuot/fiscal-mdfe` | MDF-e (em evolução) |
| `usesuot/fiscal-laravel` | Bridge opcional (DI, config) |
| `usesuot/fiscal` | Meta-pacote |

## Princípios

1. API padrão: `$cte->…->handle()` — [API pública](/fiscal/api-publica).
2. Rejeição fiscal é **Result**, não exception.
3. Host persiste, agenda retry e idempotência — o SDK não.
4. Não declare production-ready sem schemas, fixtures, ciclo de vida e homologação.
