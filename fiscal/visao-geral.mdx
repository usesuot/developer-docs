Biblioteca PHP **framework-agnostic** para documentos fiscais eletrônicos de transporte. CT-e (57), MDF-e (58) e CT-e OS (67) expõem API pública `$…->handle()` com o mesmo modelo de Result.

<Note>
Ler a documentação não exige licença. Instalar via Composer a partir de `packages.suot.dev` exige organização, licença e token.
</Note>

## Pacotes

| Pacote | Função |
|---|---|
| `suot/fiscal-core` | Tipos, VOs, Result, retry |
| `suot/fiscal-sefaz` | Autorizadores e comunicação SEFAZ |
| `suot/fiscal-cte` | CT-e (API pública `Application\Cte`) |
| `suot/fiscal-cte-os` | CT-e OS modelo 67 (API pública `Application\CteOs`) |
| `suot/fiscal-mdfe` | MDF-e modelo 58 (API pública `Application\Mdfe`) |
| `suot/fiscal-laravel` | Bridge opcional (DI, config) |
| `suot/fiscal` | Meta-pacote |

## Princípios

1. API padrão: `$cte->…->handle()`, `$mdfe->…->handle()` e `$cteOs->…->handle()` — [API pública](/fiscal/api-publica).
2. Rejeição fiscal é **Result**, não exception.
3. Host persiste, agenda retry e idempotência — o SDK não.
4. Não declare production-ready sem schemas, fixtures, ciclo de vida e homologação.
