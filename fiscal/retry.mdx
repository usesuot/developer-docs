O SDK **orienta** o retry (`RetryPolicy` + `RetryMode`). O **host** agenda, trava, consulta e reenvia.

## Modos

```php
use Suot\Fiscal\Core\Result\RetryMode;

$mode = $result->retryPolicy()?->mode;

// RetryMode::Never
// RetryMode::Immediate
// RetryMode::AfterDelay
// RetryMode::AfterCorrection
// RetryMode::QueryBeforeRetry
```

| Modo | Ação típica no host |
|---|---|
| `Never` | Não reenviar; tratar como definitivo |
| `Immediate` | Reenviar com cuidado (mesmo intent, novo operationId se necessário) |
| `AfterDelay` | Esperar e tentar de novo |
| `AfterCorrection` | Corrigir dados e remontar `$cte->issue()…` com novo `forOperation` |
| `QueryBeforeRetry` | Consultar por chave/protocolo **antes** de emitir de novo |

## O host executa

```php
use Suot\Fiscal\Core\Exception\InfrastructureException;
use Suot\Fiscal\Core\Result\RetryMode;

$policy = $result->retryPolicy();

if ($policy === null || ! $policy->isAllowed()) {
    // Host: definitivo — não reenviar
    return;
}

match ($policy->mode) {
    RetryMode::QueryBeforeRetry => (function () use ($cte, $chaveSuspeita): void {
        try {
            $query = $cte->byAccessKey()
                ->forOperation('reconcile-1')
                ->inHomologation()
                ->accessKey($chaveSuspeita)
                ->handle();
        } catch (InfrastructureException $e) {
            $failure = $e->toFailure('query');
            // Host: alertar; filar consulta conforme $failure->retryPolicy
            throw $e;
        }

        if ($query->isFound()) {
            // Host: reconciliar estado; não reemitir
            return;
        }

        // Host: só então decidir reemissão com novo forOperation
    })(),
    RetryMode::AfterCorrection => /* remontar issue() com dados corrigidos + novo forOperation */,
    RetryMode::AfterDelay, RetryMode::Immediate => /* filar com delay / imediato */,
    RetryMode::Never => /* não fazer nada */,
};
```

<Warning>
Não existe `RetryPolicy::execute()` no SDK. Não reenvie às cegas após timeout ambíguo — use `QueryBeforeRetry`.
</Warning>
