## Como funciona

A SUOT fornece SDKs independentes que a sua aplicação hospedeira consome diretamente. Não existe uma API REST intermediária — você instala os pacotes via Composer, configura credenciais e chama os handlers tipados.

```
Sua aplicação → [Laravel Bridge opcional] → SDK → SEFAZ / Serviço regulatório
```

<Note>
  A coordenação das obrigações fiscais e regulatórias pertence à sua aplicação. Os SDKs executam as operações; a sua lógica de negócio decide quando e como chamá-los.
</Note>

## Primeiros passos

<Steps>
  <Step title="Acesse o Developer Portal">
    Crie sua organização em [developers.suot.dev](https://developers.suot.dev) e gere seu token Composer. A licença é vinculada à organização.
  </Step>
  <Step title="Configure o registry Composer">
    Adicione `packages.suot.dev` como repositório Composer autenticado com seu token pessoal ou de CI/CD.
  </Step>
  <Step title="Instale o SDK desejado">
    Use `composer require` para instalar os pacotes do Fiscal SDK, do ANTT SDK ou de ambos — são independentes.
  </Step>
  <Step title="Faça sua primeira operação">
    Construa um Command, passe para o Handler e inspecione o Result. Autocomplete e PHPDoc em pt-BR guiam cada etapa.
  </Step>
</Steps>

## Por que SUOT?

<CardGroup cols={2}>
  <Card title="Tipagem total" icon="code">
    PHPStan máximo, zero métodos mágicos. Autocomplete funciona em qualquer IDE sem configuração extra.
  </Card>
  <Card title="Pronto para IA" icon="robot">
    PHPDoc em pt-BR, tipos explícitos e named arguments permitem que agentes de IA como Copilot e Claude Code gerem código correto na primeira tentativa.
  </Card>
  <Card title="Laravel ou framework-agnostic" icon="laravel">
    Use diretamente em qualquer projeto PHP 8.4+ ou ative o Laravel Bridge para injeção de dependência e configuração via `config/`.
  </Card>
  <Card title="Resultado ou exceção — sem ambiguidade" icon="shield-check">
    Rejeições esperadas retornam um `Result`. Falhas técnicas lançam `InfrastructureException`. Nunca um array anônimo.
  </Card>
</CardGroup>
