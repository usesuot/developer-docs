## Como funciona

Não há API REST intermediária. Você consome o SDK na sua aplicação:

```
Sua aplicação → [Laravel Bridge opcional] → Fiscal SDK → SEFAZ / serviço regulatório
```

<Note>
  A documentação é pública. Conta e licença não são pré-requisito para **avaliar** a API. Licença e token Composer entram quando você for **instalar** os pacotes no seu projeto.
</Note>

## Mental model (CT-e)

Monte a intenção no client e encerre em `handle()`:

```php
use Suot\Fiscal\Core\Exception\InfrastructureException;
use Suot\Fiscal\Core\Result\RetryMode;

try {
    $result = $cte->issue()
        ->forOperation('ot-000123')
        ->inHomologation()
        // … issuedBy, from, to, carrying, …
        ->handle();
} catch (InfrastructureException $e) {
    $failure = $e->toFailure('issue');
    // Host: alertar; filar retry conforme $failure->retryPolicy (timeout ambíguo → QueryBeforeRetry)
    throw $e;
}

if ($result->isAuthorized()) {
    // Host: persistir accessKey, protocolNumber, submittedDocumentXml, responseXml, receivedAt, operationId
}

if ($result->isRejected()) {
    $failure = $result->failure();
    $policy = $result->retryPolicy();
    match ($policy?->mode) {
        RetryMode::AfterCorrection => /* remontar issue() com dados corrigidos + novo operationId */,
        RetryMode::QueryBeforeRetry => /* byAccessKey() antes de reemitir */,
        RetryMode::AfterDelay, RetryMode::Immediate => /* filar conforme política */,
        RetryMode::Never, null => /* definitivo; não reenviar */,
    };
}
```

Rejeição fiscal é `Result` (`isRejected()`, `failure()`), não exception. Falha de infraestrutura (rede, certificado) é exception. O **host** persiste XML/chave e executa retry — o SDK só orienta.

## Por que SUOT?
